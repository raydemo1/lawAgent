# Connect Frontend to Review API as a Single-User Workbench

LawAgent Phase 2 turns the frontend mock into a single-user review workbench connected to the real review API and evaluation output. It should let a user submit a question and material, optionally upload a Docling-parsed PDF/DOCX beta input, view extracted review facts, structured review results, evidence status, second-retrieval status, grouped citations, missing information, recommended actions, and eval metrics or bad cases.

Phase 2 does not build multi-user auth, team collaboration, full history search, knowledge-base administration, parser-quality correction tools, prompt debugging, or chunk debugging screens.
